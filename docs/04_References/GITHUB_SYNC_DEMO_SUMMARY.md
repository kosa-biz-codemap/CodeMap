# chroma-core/github-sync-demo 요약 및 분석

**chroma-core/github-sync-demo**는 사용자가 AI를 사용하여 **어느 GitHub 레포지토리와도 대화**할 수 있게 해주는 데모 프로젝트입니다. Chroma Sync API를 사용하여 레포지토리 데이터를 자동으로 색인하고, 이를 바탕으로 코드베이스에 대한 AI 질의응답 기능을 제공합니다.

## 📌 핵심 요약 (Core Highlights)

- **AI 코드베이스 대화 (Chat with Codebase)**: 사용자가 GitHub 레포지토리의 내용을 AI와 대화하며 파악할 수 있도록 돕습니다.
- **자동화된 색인 (Automated Indexing)**: Chroma Sync API를 활용하여 레포지토리 검색 시 자동으로 인덱싱을 수행합니다.
- **Git 트리 인식 (Git-tree aware)**: 마지막 동기화 이후 변경된 파일만 식별하여 효율적으로 재인덱싱합니다.
- **Chroma 기반 벡터 검색**: 인덱싱된 데이터는 표준 Chroma 컬렉션으로 구성되어, 강력하고 빠른 유사도 검색을 지원합니다.

## ⚙️ 기술 스택
- **프론트엔드/풀스택 프레임워크**: Next.js, TypeScript
- **데이터베이스 및 AI 인프라**: Chroma (오픈소스 AI 애플리케이션 데이터베이스)

## 🏗️ 주요 아키텍처 및 파이프라인 흐름

### 1. 저장소 동기화 및 인덱싱 파이프라인
1. 사용자가 GitHub 레포지토리 URL 등을 입력하여 인덱싱을 요청합니다.
2. 애플리케이션이 레포지토리 파일들을 로드한 뒤, 적절한 크기로 청킹(Chunking)하고 임베딩을 생성합니다.
3. 변환된 문서와 임베딩 벡터가 Chroma 컬렉션에 삽입(Insert)됩니다.

### 2. 점진적 업데이트 최적화 (Incremental Update)
- 전체 레포지토리를 매번 처음부터 다시 처리하지 않습니다.
- Git 트리를 인식(Git-tree aware)하여 변경 사항이 발생한 특정 파일에 대해서만 임베딩을 다시 수행하여 리소스를 절약합니다.

### 3. 질의응답 검색 흐름 (RAG 파이프라인)
- 저장된 Chroma 컬렉션을 대상으로 사용자의 질문과 관련된 코드 조각들을 검색합니다.
- 검색된 문맥(Context)을 기반으로 AI 모델이 분석하여 최종적인 답변을 사용자에게 제공합니다.

---

> **💡 CodeMap 프로젝트 적용 관점**
> Chroma를 활용해 대규모 외부 데이터를 AI의 문맥 창으로 연결하는 실용적인 사례입니다. 특히 전체를 재스캔하지 않고 **Git 트리 변화를 감지하여 변경된 파일만 동기화하는 전략**은 RAG 기반 분석 도구를 최적화할 때 적극 참고할 만한 아키텍처입니다.
