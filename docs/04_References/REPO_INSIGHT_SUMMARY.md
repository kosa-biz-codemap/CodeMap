# CronusL-1141/repo-insight 요약 및 분석

**CronusL-1141/repo-insight**는 Python 레포지토리를 자율적으로 분석하고 시각화하는 데 특화된 **멀티 에이전트(Multi-Agent) 분석 도구**입니다. 제한된 시간 내에 효율적으로 코드를 분석하여 인사이트를 제공하는 데 중점을 둡니다.

## 📌 핵심 요약 (Core Highlights)

- **멀티 에이전트 아키텍처 (Multi-Agent Architecture)**: 4개의 에이전트가 동시에 협력하며, 3가지 서로 다른 멀티 에이전트 메커니즘을 활용해 대상 레포지토리를 다각도로 분석합니다.
- **실행 시간 제한 (Performance Constraints)**: 120초(2분)라는 엄격한 분석 예산(Budget) 내에서 작동하도록 설계되어, 토큰 효율적이고 신속한 분석을 보장합니다.
- **인터랙티브 히트맵 결과 (Output)**: 분석 결과를 바탕으로 인터랙티브 히트맵 리포트를 생성하여 레포지토리의 주요 구조와 인사이트를 시각적으로 쉽게 파악할 수 있도록 제공합니다.
- **에이전트 생태계 연동**: 개발자의 다른 프로젝트인 AI-company(멀티 에이전트 팀 OS)와 유사한 철학을 공유하며, 에이전트 주도의 자율적 코드 분석 워크플로우를 구현합니다.

## ⚙️ 기술 스택
- **주요 언어 (Primary Language)**: Python
- **구조/프레임워크**: 병렬 처리를 지원하는 멀티 에이전트 프레임워크 (Multi-Agent Framework)

## 🏗️ 주요 아키텍처 및 파이프라인 흐름

### 1. 동시성 에이전트 파이프라인 (Concurrent Agent Pipeline)
타겟 레포지토리가 주어지면 에이전트 기반 분석이 병렬로 실행됩니다.
1. 4개의 동시 실행 에이전트(Concurrent Agents)가 각자의 역할에 맞게 코드베이스에 접근합니다.
2. 3가지 고유한 멀티 에이전트 메커니즘을 통해 코드를 교차 검증하고 분석 데이터를 수집합니다.

### 2. 타임박스 최적화 전략 (Time-Boxed Optimization)
- 모든 분석 과정은 **120초 실행 예산(Execution Budget)** 이라는 엄격한 규칙 아래 수행됩니다.
- API 호출 및 컨텍스트 로드를 최적화하여 시간 내에 레포지토리의 핵심 인사이트를 추출하도록 설계되었습니다.

### 3. 히트맵 시각화 파이프라인 (Heatmap Synthesis)
에이전트들의 코드 분석 및 취합이 완료되면, 수집된 데이터를 바탕으로 최종 리포트가 생성됩니다. 이 결과물은 일반적인 텍스트 요약이 아닌 **인터랙티브 히트맵(Interactive Heatmap)** 형태로 제공되어 직관적인 구조 파악을 돕습니다.
