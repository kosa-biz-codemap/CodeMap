# jurasofish/mcpunk 요약 및 분석

**mcpunk**는 Model Context Protocol(MCP)을 기반으로 구축된 오픈소스 서버로, 복잡한 코드베이스를 벡터 임베딩 없이도 지능적으로 탐색하고 이해할 수 있게 해주는 **Roaming RAG** 도구입니다.

## 📌 핵심 요약 (Core Highlights)

- **임베딩 없는 Roaming RAG**: 기존의 벡터 임베딩 기반 검색 대신 구조적 접근법을 사용하여 코드베이스를 탐색합니다.
- **논리적 청킹 (Logical Chunking)**: 프로젝트 디렉토리 내의 파일들을 단순 텍스트가 아닌 논리적인 단위(Chunk)로 세분화하여 LLM이 분석하기 쉽게 제공합니다.
- **도구 기반 탐색 (Tool-Based Interaction)**: LLM에게 특정 청크나 파일을 텍스트 기반으로 쿼리할 수 있는 도구를 제공하여, 코드 내 API 사용 일관성 확인이나 논리적 오류 탐지 등을 수행할 수 있습니다.
- **범용적인 MCP 표준 준수**: AI 애플리케이션(MCP Host)과 외부 데이터(코드베이스)를 연결하는 표준화된 MCP 서버로 동작하여 높은 호환성을 가집니다.

## ⚙️ 기술 스택

- **서버 구현 언어**: Python
- **프로토콜 및 연동**: Model Context Protocol (MCP) SDK
- **파일 시스템 인터페이스**: 로컬 파일 시스템 파싱 및 검색

## 🏗️ 주요 아키텍처 및 파이프라인 흐름

### 1. 코드 발견 및 분석 파이프라인 (Code Discovery)
사용자 또는 LLM이 코드베이스에 대한 질문을 하면 다음 과정이 실행됩니다.
1. 대상 프로젝트 디렉토리의 파일들을 분석하여 논리적 청크(Logical Chunks)로 분할합니다.
2. LLM이 서버가 제공하는 탐색 도구를 호출하여 텍스트 기반 쿼리를 실행합니다.
3. 관련 청크와 파일의 맥락을 찾아 LLM에 제공함으로써 코드의 논리적 일관성을 확인하거나 복잡한 코드를 설명합니다.

### 2. Model Context Protocol (MCP) 기반 구조
- **MCP Host**: Claude Desktop이나 IDE 등과 같은 AI 애플리케이션이 오케스트레이터 역할을 수행합니다.
- **MCP Server**: mcpunk 프로세스가 코드베이스 리소스와 검색 도구를 Host(클라이언트)에 노출하여 실시간 상호작용을 가능하게 합니다.

---

> **💡 프로젝트 적용 관점**
> MCPunk는 **벡터 DB나 임베딩 구축 없이 논리적 청킹과 도구 기반 탐색(Tool-Based Search)만으로 RAG를 구현**한 흥미로운 레퍼런스입니다.
> AI 모델 스스로가 제공된 검색 툴을 이용해 파일 시스템을 순회(Roaming)하며 필요한 코드 컨텍스트를 찾아내는 방식은 복잡한 프로젝트 온보딩이나 AI 코드 어시스턴트 구조를 설계할 때 매우 훌륭한 벤치마크가 될 수 있습니다.
