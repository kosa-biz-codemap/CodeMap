# danielefavi/ai-codebase-assistant 요약 및 분석

**ai-codebase-assistant**는 로컬 환경에서 안전하게 복잡한 코드베이스를 이해하고 탐색할 수 있도록 돕는 **프라이버시 중심의 AI 코드 어시스턴트 API 서버**입니다.

## 📌 핵심 요약 (Core Highlights)

- **프라이버시 최우선 (Privacy-Focused)**: 로컬 **Ollama** 모델을 활용하여 코드 데이터가 외부 서버로 전송되지 않고 사용자의 로컬 환경에 안전하게 유지됩니다.
- **RAG (Retrieval-Augmented Generation) 기반**: 검색 증강 생성 방식을 사용하여 질문과 관련된 코드 스니펫 및 요약본을 신속히 검색하고, 이를 바탕으로 문맥을 인지한 정확한 답변을 제공합니다.
- **자동화된 코드 스캐닝**: 프로젝트를 스캔하여 새 파일이나 수정된 파일을 지능적으로 감지하고, 코드를 분석하기 좋은 논리적인 청크(Chunk)로 분할합니다.
- **지식 기반 검색 (Knowledge Graph)**: 분할된 코드 청크의 요약을 생성한 뒤, 원본 코드와 함께 로컬 벡터 데이터베이스에 저장하여 검색 효율성을 극대화합니다.

## ⚙️ 기술 스택
- **백엔드/런타임**: Node.js, TypeScript
- **AI/LLM**: LangChain.JS, Ollama (Local LLMs)
- **인프라 및 데이터베이스**: Docker (서비스 컨테이너화), MySQL, Vector Database

## 🏗️ 주요 아키텍처 및 파이프라인 흐름

### 1. 스캔 및 인덱싱 (Scanning & Indexing)
에이전트가 지정된 프로젝트 디렉토리를 주기적 또는 요청 시점에 스캔합니다. 변경 사항을 확인한 후 파일 코드를 논리적인 청크로 나누어 시스템이 처리할 수 있는 단위로 준비합니다.

### 2. 요약 및 임베딩 저장 (Summarization & Storage)
로컬 LLM을 통해 분리된 코드 청크들의 역할을 요약(Summarization)합니다. 이후 해당 요약과 코드 원본 데이터를 벡터화(Embedding)하여 로컬 벡터 데이터베이스와 메타데이터 저장소(MySQL 등)에 연동해 저장합니다. 

### 3. 사용자 질의 처리 (Querying Pipeline)
준비된 벡터 인덱스를 기반으로 사용자가 API를 통해 질문을 던지면, RAG 파이프라인이 작동하여 가장 관련성이 높은 코드 스니펫들을 도출합니다. 이렇게 수집된 맥락을 로컬 LLM에 제공하여 질문에 부합하는 최종 답변을 생성합니다.
