# 403errors/repomind 요약 및 분석

**403errors/repomind**는 퍼블릭 GitHub 레포지토리와 개발자 프로필을 쉽게 분석하고 이해할 수 있도록 돕는 **AI 기반의 오픈소스 플랫폼**입니다. 코드를 직접 읽는 대신 AI와 대화하며 구조를 파악할 수 있게 해주는 도구입니다.

## 📌 핵심 요약 (Core Highlights)

- **Agentic CAG (Context-Augmented Generation)**: 기존의 RAG(단편적인 벡터 임베딩 청크 기반 검색) 방식의 한계를 극복하기 위해, AI가 먼저 어떤 파일이 필요한지 판단(File Selection)하고 **전체 코드를 온전한 문맥으로 유지한 채 로드**하여 답변을 생성합니다.
- **시각적 이해 제공 (Visual Understanding)**: 단순한 텍스트 답변을 넘어, 코드베이스의 아키텍처 맵과 흐름도(Mermaid 활용)를 생성하여 프로젝트 온보딩 속도를 높여줍니다.
- **제로 셋업 (Zero Setup)**: 별도의 환경 설정이나 클론 과정 없이 퍼블릭 레포지토리 URL이나 프로필을 입력하기만 하면 즉시 분석을 시작합니다.
- **실시간 진행 상태 (Live Step Feedback)**: AI가 파일을 선택하고, 읽고, 답변을 준비하는 모든 과정을 실시간 텍스트 피드백으로 투명하게 보여줍니다.
- **보안 스캐닝 내장 (Security Scanning)**: OSV API를 활용한 실시간 취약점 체크와 결정론적 보안 엔진을 통해 레포지토리의 보안 스캔 기능(Quick/Deep)을 기본 제공합니다.

## ⚙️ 기술 스택
- **프론트엔드/백엔드**: Next.js 16 (Browser-first 경험 제공)
- **AI/LLM**: Google Gemini (Gemini 3 Flash Preview 등 활용)

## 🏗️ 주요 아키텍처 및 파이프라인 흐름

### 1. Query & Context 파이프라인
사용자가 레포지토리를 검색하면 다음과 같은 Agentic 파이프라인이 실행됩니다.
1. 메타데이터 및 파일 트리(File Tree) 수집
2. **AI 파일 선택기(GEMINI_FILE_SELECTOR_MODEL)**가 질문에 필요한 핵심 파일 25개~50개를 우선 선별
3. 선별된 파일을 GitHub API를 통해 가져와 파일 트리 구조와 함께 Context로 구성
4. 구성된 맥락을 바탕으로 생성 모델(Lite/Thinking)이 답변 및 아키텍처 맵 생성

### 2. 캐싱 및 최적화 전략 (Caching Strategy)
- 빠른 응답 속도를 위해 **SHA 기반의 파일 캐시**를 사용합니다. 
- GitHub API 호출을 최소화하기 위해 2MB 이하의 페이로드는 Gzip으로 압축하여 KV 스토어에 보관하며, '질문-선택된 파일 매핑'과 '질문에 대한 최종 답변'은 24시간 동안 캐싱 처리합니다.

### 3. 보안 스캔 파이프라인 (Security Verification)
스캔이 시작되면 종속성 및 스코프 내 파일들을 수집하고, 정적 분석 보안 엔진과 외부 취약점 데이터베이스(OSV API)를 동시에 확인하여, 필요시 AI의 검증을 거쳐 최종 리포트를 생성합니다.
