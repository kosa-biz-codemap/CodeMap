# 🚀 PR 규칙 (Pull Request Convention)

* **PR 제목**: `{커밋 유형}: {PR 제목}` (예: `FEAT: 로그인 폼 UI 컴포넌트 추가`)
* **단일 명세 원칙**: 하나의 PR에는 방대한 범위를 담지 않고, 반드시 **개별 API 명세서 단위별** 또는 **세부 기능 단위별로 쪼개서** 올려주세요. (예: `API-001/003/005`를 한 번에 묶지 말고 `API-001` 단일 PR로 분리)
* **리뷰어 할당**: PR 생성 시 `Reviewers` 항목에 **본인 외 나머지 팀원 모두**를 반드시 포함시켜 주세요.
* **작업 흐름**:
  1. `main` 브랜치에서 최신 버전을 Pull 받은 후 새로운 기능 브랜치 생성
  2. 작업 완료 후 로컬 `main`을 머지하여 충돌(Conflict) 해결
  3. 충돌 없는 코드를 원격 저장소에 Push 후 PR 생성
