🔍 Problem Definition
왜 CodeMap이 필요한가?

낯선 코드베이스에 합류하는 개발자의 가장 큰 페인 포인트

1
온보딩 장벽
README.md 만으로는 전체 의존성 및 설계 흐름을 유추하기 어렵습니다. 새 팀에서 "어디서부터 봐야 하는지"를 아무도 알려주지 않습니다.
2
막대한 시간 소모
엔트리포인트 탐색 → 로컬 환경 구동 → 핵심 비즈니스 로직 파악까지 수 시간에서 수 일이 소요됩니다.
3
기존 챗봇의 한계
단순 코드 Q&A 챗봇은 조각난 코드 스니펫만 반환합니다. 프로젝트 전체의 큰 그림(Architecture)을 통합 제공하지 못합니다.
💡
CodeMap의 해결책
단순 질의응답을 넘어, 코드베이스 분석 + RAG 파이프라인을 융합한 온보딩 마스터 리포트 자동 생성.

데이터베이스 & RAG 전략
1
정적 RAG + Agentic CAG 하이브리드
임베딩 벡터 DB로 후보 파일군을 1초 내에 좁힌 뒤, 에이전트가 직접 탐색 도구를 호출해 정밀 분석. 거대 레포에서의 탐색 속도와 정확도를 동시에 확보합니다.
2
계층형 트리 RAG (Tree-Based RAG)
개별 파일 임베딩 → LLM으로 폴더 단위 자연어 요약 생성 → 그 요약본을 다시 임베딩(Bottom-up). 벡터 평균이 아닌 자연어 요약 후 재임베딩으로 의미 훼손을 방지합니다.
3
AST 기반 시맨틱 코드 청킹
단순 문자 수 기준 분할의 한계를 넘어, 함수/클래스/라우터 단위로 코드를 분할합니다. 파일 경로, 함수명 등 풍부한 메타데이터를 함께 저장합니다.
4
의존성 그래프 스키마 (pgvector)
폴더 구조를 무시하는 코드 참조 관계를 JSONB 문자열 배열로 저장. imports, imported_by를 PostgreSQL JOIN과 WITH RECURSIVE로 방사형 지식 그래프를 구성합니다.

🧠 Agent LLM
GPT-4o
Supervisor Agent의 계획 수립, Search Worker의 Query Rewrite/Semantic Search, Final Answer Agent의 근거 기반 답변 생성을 담당합니다.

코드와 문서를 수학적 벡터 공간으로 변환. 다국어 정보 검색 평가(MIRACL)에서 압도적 점수로 한글 주석 + 영문 소스코드 혼재 환경에 최적화.
✅ 차원 축소해도 성능 유지되는 마트료시카 표현 학습 지원
💰 Cost Optimizer
GPT-4o-mini
Map-Reduce 요약 파이프라인 및 초기 데이터 전처리. 대량 텍스트 문서화 단계에서 발생할 막대한 API 비용을 절감합니다.
